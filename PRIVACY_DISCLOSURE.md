# Network Activity & Privacy Disclosure

## Overview

The Quartermaster is a community marketplace mod for SPT (Single Player Tarkov). It connects to external cloud services (Google Firebase) to enable a shared, cross-server marketplace where players can buy and sell items. The mod also synchronises community-created contracts from an external workshop website, and optionally links a Discord account for community voting.

**All network activity uses HTTPS (TLS-encrypted) connections.**

---

## Destination Servers

### 1. Google Firebase Authentication
- **URL:** `https://identitytoolkit.googleapis.com/v1/accounts:signUp`
- **URL:** `https://securetoken.googleapis.com/v1/token`
- **Purpose:** Authenticates the server anonymously with Firebase to obtain an access token for Realtime Database and Firestore access.
- **Data sent:** Firebase API key (public, non-sensitive). No player profile data, no personal information. The response contains an anonymous UID and a temporary auth token.
- **Frequency:** On server startup, and when the auth token expires (approximately every hour).
- **Privacy impact:** A random anonymous ID is generated by Google. This ID is used to identify your server's listings in the marketplace. It is not linked to your SPT profile, your real identity, or your IP address by the mod.

### 2. Firebase Realtime Database (RTDB)
- **URL:** `https://spt-the-quartermaster-default-rtdb.europe-west1.firebasedatabase.app/`
- **Purpose:** Stores and retrieves marketplace listings (items that players have listed for sale), listing states, buy filters, listing quantity limits, and a catalogue version counter.
- **Data sent (when you sell an item):**
  - Item template ID (e.g. `58864a4f2459770fcc257101`)
  - Item name and short name
  - Item tree (full item structure including attachments, mods, stack size)
  - Base price and market price (in-game roubles)
  - Quality multiplier
  - Whether the item is vanilla or modded
  - Seller hash (a SHA-256 hash of your profile ID — not your actual profile ID)
  - Server ID (your SPT server's root folder name)
  - Created and expiry timestamps
  - Listing status (active/sold/expired)
  - Quantity being sold
- **Data received (when browsing the trader):** All active listings from all participating servers, including the above fields from other players' listings.
- **Frequency:** On server startup, then every 30 minutes (configurable via Firestore) for a full refresh. Individual reads/writes occur when you sell or buy items.
- **Privacy impact:** Your SPT profile ID is never sent directly — it is hashed with SHA-256 before being transmitted. The item data you sell is visible to other servers who use the marketplace. Your server's root folder name is sent as a "Server ID" to identify which server listed the item. No real-world personal information (name, email, IP) is collected or transmitted by the mod.

### 3. Google Firestore
- **URL:** `firestore.googleapis.com` (via Google Cloud Firestore SDK)
- **Purpose:** Stores and retrieves:
  - **Backend config** (`quartermaster_config/contract_config`): Markup percentages, workshop API URL, SPT version
  - **Mod version gate** (`quartermaster_config/mod_version`): Minimum required mod version
  - **Listing config** (`quartermaster_config/listing_config`): Listing duration and refresh cooldown settings
  - **Contract definitions** (`quartermaster_contracts`): Community contract quest definitions
  - **Contract submissions** (`quartermaster_submissions`): Pending community contract submissions awaiting approval
  - **Contract schedule** (`quartermaster_schedule`): When contracts should activate
  - **Contract votes** (`quartermaster_votes`): Community voting data on contract submissions
  - **Meta** (`quartermaster_meta`): Contract version counter
- **Data sent:** Contract quest data (objectives, rewards, descriptions, messages), contract metadata (author name, approval status, voting results), and config values. No player profile data or item data is stored in Firestore.
- **Frequency:** On server startup, and as a fallback if the contract caching server (see below) is unreachable. Under normal operation, the mod no longer reads Firestore directly for contract data — it reads from the caching server instead.
- **Privacy impact:** Contract authors' display names and UIDs are stored. No SPT player profile data is sent to Firestore. The mod version check on startup sends only your installed mod version number (compared locally, not transmitted).

### 4. Contract Caching Server (HTTP)
- **URL:** `http://144.21.60.21/contracts/data.json`
- **Purpose:** Serves a bundled JSON file containing all current contract definitions, schedule entries, submissions, and backend config. A central server reads Firestore every 5 minutes and writes this file, so individual players' servers don't need to read Firestore directly.
- **Data sent:** None. This is a read-only HTTP GET request — no data is transmitted from your machine to the server.
- **Data received:** A JSON document containing contract definitions, schedule entries, submissions, and backend config. This data is read into memory only — **no file is written to your disk**.
- **Frequency:** On server startup, then every 5 minutes (cached locally for 5 minutes between fetches).
- **Privacy impact:** None. Your IP address is visible to the server as with any HTTP request, but no personal or game data is sent. The connection is unencrypted (HTTP, not HTTPS), but no sensitive data is transmitted in either direction. If this server is unreachable, the mod automatically falls back to reading from Firestore directly.
- **Note:** This server also performs marketplace cleanup (deleting expired and sold listings from RTDB) on behalf of all players, which was previously done by each player's server individually. This reduces bandwidth usage for all players by approximately 90%.

### 5. Workshop API (External Website)
- **URL:** Configured in backend config (default: `https://serenity-workshop.netlify.app/api/contract-list`)
- **Purpose:** Syncs approved community contracts from the workshop website into the mod's Firestore database.
- **Data sent:** HTTP GET requests with a query parameter `?type=active|contracts|submissions`. No POST data, no player data, no authentication tokens.
- **Data received:** JSON responses containing contract definitions, schedule entries, and submission data from the workshop.
- **Frequency:** On server startup, then every 5 minutes.
- **Privacy impact:** This is a read-only fetch. No personal or game data is sent to the workshop website. Your IP address is visible to the website as with any HTTP request.

### 6. Discord Linking & Community Voting (Workshop API)

- **URL:** `https://serenity-workshop.netlify.app/.netlify/functions/` (endpoints: `link-request`, `link-status`, `contract-vote`)
- **Purpose:** Allows you to link your Discord account so you can upvote or downvote pending community contract submissions.
- **Data sent (linking):**
  - A one-time UUID generated by the mod (`uuid`) is sent to `link-request` to start the link flow.
  - Polling `link-status` with the same UUID to retrieve the resulting Firebase custom token once you complete Discord OAuth on the workshop website.
- **Data sent (voting):**
  - The submission ID you are voting on and whether your vote is an upvote or downvote.
  - Your Firebase `id_token` is sent in the `Authorization: Bearer` header so the backend can identify your Discord account.
- **Data received:** A temporary link code for opening the Discord OAuth page in a browser, and a Firebase authentication token once linking is complete.
- **Frequency:** Linking is a one-time user-initiated action. Votes are sent only when you press the upvote/downvote buttons in the community panel.
- **Data stored locally:** `community_auth.json` is saved in the mod's `TheQuartermaster/` folder. It contains your generated mod UUID, Firebase `id_token`, Discord display name, and token expiry time.
- **Privacy impact:** Your Discord account is used only for identity and voting. The mod does not read your Discord messages, friends, or servers. Your Discord user ID is stored server-side in Firestore and linked to your workshop profile. You can unlink at any time from the community panel, which clears the local auth file.

## Data Retention

- **Active listings** are stored in RTDB until they expire (configurable) or are sold.
- **Sold listings** are moved to a `listings/sold/` path and automatically deleted 5 minutes after being marked as sold.
- **Expired listings** are automatically deleted from RTDB.
- **Contract data** in Firestore persists until manually removed by an admin or expired by the scheduler.
- **Local cache files** (`lastRefresh.json` and `community_auth.json`) store a timestamp of the last RTDB refresh and your Discord link token (UUID, Firebase id_token, Discord display name, and token expiry) on the local server.

---

## What Is NOT Collected or Transmitted

- Your SPT profile ID (only a SHA-256 hash of it is sent)
- Your real name, email, or any personal information
- Your SPT account password or session tokens
- Your game save data or profile progress
- Your IP address (not collected by the mod; standard HTTPS connections are used)
- Your installed mods list (only whether listed items are "vanilla" or "modded" is indicated)
- Your Discord messages, friend list, or server memberships
- Your Discord password or email address

---

## How to Disable Network Activity

The mod can be disabled entirely by setting `modEnabled: false` in `config.json`. When disabled, no network connections are made. The trader will not appear in the game.

## Summary for Non-Technical Users

The Quartermaster mod connects to Google's cloud services to share a marketplace between different SPT servers. When you sell an item to the Quartermaster trader, details about that item (what it is, its price, its attachments) are uploaded to a shared database so that players on other servers can buy it. When you buy an item, you're buying something another player on a different server listed for sale.

Your actual SPT profile is never sent anywhere — only a scrambled hash of your profile ID that can't be reversed to identify you. No personal information (name, email, password) is ever collected or transmitted. All connections are encrypted.

The mod also periodically downloads community-created quest/contract content from a workshop website. This is a one-way download — no game data is sent to the website. If you choose to "Link Discord" in the community panel, you can vote on community contract submissions; this sends your vote and an identity token to the workshop so it knows which Discord account to record the vote against. You can unlink at any time to clear the local auth file.

As of v1.2.0, contract data is fetched from a dedicated caching server instead of each player reading the database directly, and you can optionally link Discord to vote on community contract submissions. The contract cache is a read-only request — no game data is sent from your machine. The Discord linking and voting flow sends only the data described in the "Discord Linking & Community Voting" section above. The caching server also handles marketplace cleanup (removing expired and sold listings) on behalf of all players, which reduces bandwidth usage for everyone. If the caching server is ever unavailable, the mod automatically falls back to reading from the database directly.

You can turn off all network activity by disabling the mod in its config file.
